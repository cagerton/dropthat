sha2loader

Do you trust bootstrapcdn? cdnjs? aspnetcdn? googlecdn?

You shouldn't need to.

Sha2loader acts as an async asset loader and integrity verifier for scripts / css / images / or fonts that are hosted on servers outside your control. It lets you verify that each asset is exactly what you expected before using it. This could help mitigate the impact of public-CDN compromise or just reduce the number of your own servers that you need to trust.

How it works:

Once you've loaded sha2loader (preferably compressed to with sjcl and hosted on your own server) you can include tags with the data-sha and data-href/url attributes. You can also include a template to render css with blob-urls (for font loading). You'll need to specify urls that allow cross origin resource shraring.

This page loads bootstrap.css, d3.js, jquery.js, font-awesome icons, and a jpg image using several distinct CDNs.

Under the hood

The loader.js code is super simple. DOM modifications are picked up with a MutationObserver; any freshly inserted nodes containing the data-sha attribute are hijacked. The coresponding resources are loaded with XMLHttpRequests that have responseType='arraybuffer'. It's relatively fast and easy to feed this response type into SJCL for hashing and then to convert it in to a blob-url for use. The whole process has significantly less overhead that I originally expected.

On a practical note

Most CDNs are probably more secure than your servers, this is experimental code, and the RightWay™ to do this might involve adding native hash-checking support to the HTML spec. This is meant to be food for thought.
